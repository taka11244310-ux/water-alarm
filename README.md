# water-alarm

 1.サービス概要
   
   私が考えたサービスは、水分補給の通知アプリです。高齢者の方は、水分不足になりやすく、そうすると熱中症や認知症のリスクが高まります。このアプリを利用していただくことで、水分補給の通知が届き、飲んだ量が把握し、健康維持を助けることができます。


 2.このアイデアを考えた理由

 　2-1きっかけとなった体験

 このアイデアの元になったのが、去年の夏のことです。
 
 私は高齢者施設のデイサービスで働いているのでが、そこに利用して下さっているご利用者様のお一人が熱中症になったことです。
 
 その方は、お一人暮らしで、認知症もあり、水分摂取のされることも少なく、熱中症になってしまいました。
 
 その際に、水分補給を促す何かがあれば防ぐことが出来たのではないかと思い、このアイデアを考えました。

  2-2　なぜそれが「気になった」のか

　 高齢者の方と関わる上で、毎年熱中症の予防に努めましょうという声掛けをしていますが、それでもエアコンをつけない方や水分を摂られない方はおられます。

また、高齢者の方は、喉の渇きのセンサーやトイレが近くなるのが嫌なでの理由で、水分を摂られる方が少ないので、このアプリで少しでも手助けになればいいと思い考えました。

3.課題の整理

  3-1.困りごと

   一番課題は、独居の方です。
   
   独居の方は水分補給を促してくれる人が近くにいない
   認知症の方は、より水分摂取の認識が低下する
   結果として熱中症のリスクが高くなる。
 
  3-2  本当に解決したい課題は何か

   熱中症の予防
   認知症の予防（水分不足は認知機能の低下の恐れがある）
   健康寿命

4. 想定ユーザーについて

  4-1. 想定しているユーザー

  年齢設定
    年齢：高齢者の方（65歳以上〜）
    理由：65歳から体温調整の機能が低下してくるため、水分摂取する方が少なくなる為。
  
  生活・状況設定：
    生活・状況 : 独居の方や認知症の方
    理由 :定期的な水分補給を促し、熱中症や認知症を予防できる。

  4-2. 実際に使われる可能性について

   最初に使ってくれる人は、スマホに少し慣れている高齢者やそのご家族様が両親や祖父母に使うようにしてくれる。

　4-3. ユーザーがサービスを導入して利用するまでのイメージ

　　使用して下さるきっかけ：まず、使っていみようと思うきっかけは、本人やそのご家族が高齢になり、熱中症や認知症のリスクが高くなっているという時に使用してくださるイメージ。

　　最初の一歩：自分の体重・メールアドレスの入力。
　　利用を続ける理由は：熱中症や認知症の予防。

    私が作りたいのは、「コップ1杯」「半分」「3分の1」を選ぶだけの

6.　このサービスで提供したい価値

  6-1. ユーザーの変化

   本人やご家族様が少しでも安心して生活できるサービス。

  6-2. 価値を一文で表す

    このサービスは、水分不足の高齢者が、適切な量の水分補給を行えるサービス。
    
7. このアプリで実現すること

  MVPでつくる機能

  1.体重に合わせて一日の水分量の目安を設定する。
  （計算式：　体重　✖️　30ml = 1日の目標水分量）


  2.水分摂取の記録機能
  飲んだ水分量を入力する。（高齢者の方でも容易に操作出来るように、コップ一杯・半分・3分の１を表示、そこから選択するだけの選択形式にする。）

  3.メールで水分補給の通知が来るようにする。


8. このアプリの懸念点とその対策

  懸念点①

  問題点：　個々によって就寝時間や起床時間が違う為、何時まで通知を行うかを設定をしないといけない。

  対策：初期設定で起床時間や就寝時間を選択形式で設定してもらうようにする。

  懸念点②　スマホを持っていない高齢者

  問題点：　スマホを持っていない高齢者やスマホの操作がまったくできない方には使えない

  対策：　スマホに少し慣れている高齢者をターゲットにしていく

9.今後の展開・発展の方向性
  
  ユーザーが使いづつける理由：定期的な通知がくることにより、水分摂取を習慣づける

  続けることで変わる理由：　水分補給を行うことで、その方の体調が悪化しないように予防ができる。また、その予防が健康寿命にもつながる。

  他の人との関わり：ご家族様も水分摂取の状況を見れるようにすれば、一人暮らしの方でも遠方のご家族様でも把握ができる。

  今考えている発展案

  　1.スマホに通知が来るようにする。

    2.ご家族様にも水分摂取の状況が確認できるようにする。
